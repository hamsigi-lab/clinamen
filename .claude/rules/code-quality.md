# 코드 품질 규칙

## 작성 원칙
- 함수는 하나의 일만 한다
- 변수/함수명은 역할이 명확하게 드러나도록 짓는다
- 주석은 WHY만 — 코드가 무엇을 하는지는 이름으로 표현한다
- 파일 하나가 너무 길어지면 분리를 먼저 고려한다

## 금지 사항
- `eval()`, `innerHTML`에 사용자 입력 직접 삽입 (XSS)
- API 키, 토큰, 비밀번호 하드코딩
- `console.log` 배포 코드에 잔류
- 사용하지 않는 변수, 함수, import

## 스타일
- 들여쓰기: 2 spaces
- 문자열: single quote
- 세미콜론: 사용
- CSS: 모바일 퍼스트 (min-width 기준 미디어 쿼리)
